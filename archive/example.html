    2 - 1º parte da configuração

    No terminal digite: ssh-keygen -t rsa -b 4096 -C "seu_email@example.com"

    - Quando for solicitado um local para salvar a nova chave, escolha um nome diferente do padrão `~/.ssh/id_rsa`. Por exemplo, você pode salvar como `~/.ssh/id_rsa_nova`.

    - Durante o processo, você será solicitado a fornecer uma senha. Essa etapa é opcional, mas recomendada para maior segurança.

    - Depois que a chave for gerada, adicione a nova chave ao seu agente SSH: eval "$(ssh-agent -s)"

    O comando `eval "$(ssh-agent -s)"` é usado para iniciar o agente SSH e configurar o ambiente do shell para usá-lo. Vamos entender melhor:

    1. Iniciar o agente SSH**: O `ssh-agent` é um programa que mantém suas chaves SSH carregadas na memória. Isso evita que você tenha que digitar sua senha de chave privada toda vez que usar a chave para se autenticar em um servidor ou repositório remoto.

    2. Configurar o ambiente do shell: O comando `eval "$(ssh-agent -s)"` avalia (executa) a saída do comando `ssh-agent -s`, que retorna as variáveis de ambiente necessárias para o funcionamento do agente SSH. Essas variáveis de ambiente são adicionadas ao seu ambiente de shell atual, permitindo que ele se comunique com o agente SSH recém-iniciado.

    Basicamente, esse comando prepara seu ambiente de trabalho para usar o agente SSH, facilitando o gerenciamento das suas chaves SSH durante a sessão.
